# DynamicPersona

`astrbot_plugin_DynamicPersona` 是一个 AstrBot 插件。

它会根据用户当前消息的语义，在你预先配置的人格规则中选择一个人格，并在需要时把当前对话切换到该人格对应的对话模型配置。

## 功能

- 根据消息内容动态选择人格
- 切换人格时同步切换到该人格绑定的对话模型配置
- 支持为 Selector 单独指定一个较快的模型
- 支持按会话白名单或黑名单控制生效范围
- 支持缓存最近一次人格决策，减少重复选择

## 工作方式

插件内部维护的是一组“人格规则”。

每条规则包含：

- 一个 AstrBot 人格
- 该人格的描述
- 该人格适用的场景描述
- 一个 AstrBot 已配置的对话模型条目

这里选择的不是自由输入的模型名，而是 AstrBot 配置页面中已经存在的对话模型配置。插件设置页会通过下拉框读取这些已配置条目，用户直接选择即可。

当一条消息到来时，插件会：

1. 判断当前会话是否启用该插件
2. 检查当前会话是否已经绑定 AstrBot 原生人格
3. 用 Selector 模型在规则中选择最合适的人格
4. 把当前对话切换到该人格对应的对话模型配置
5. 在真正请求 LLM 前注入该人格的 system prompt

## 安装

将插件放入 AstrBot 的插件目录，例如：

```bash
AstrBot/data/plugins/astrbot_plugin_DynamicPersona
```

然后在 AstrBot 中重载或启用插件。

## 使用前准备

在配置本插件前，请先在 AstrBot 中完成以下准备：

1. 在人格管理中创建好需要切换的 AstrBot 人格
2. 在 Provider 页面配置好需要使用的对话模型条目
3. 确认这些模型条目可以正常工作

如果你没有先配置人格或对话模型，本插件的规则将无法正确工作。

## 配置步骤

### 1. 启用插件

在插件配置中将 `enabled` 设为 `true`。

### 2. 配置 Selector 模型

`selector_provider_id` 用于指定“负责选择人格”的模型。

- 留空：使用当前会话正在使用的模型
- 建议：单独选择一个速度较快、成本较低的模型

Selector 只负责判断应该切换到哪个人格，不负责最终回答用户。

### 3. 配置注入方式

`inject_mode` 有两个选项：

- `replace`：直接用选中的人格 prompt 替换当前 system prompt
- `prepend`：把选中的人格 prompt 插到当前 system prompt 前面

通常建议使用 `replace`。

### 4. 配置缓存

`cache_ttl` 表示同一会话中，最近一次人格决策可以复用多少条消息。

- `0`：每条消息都重新选择人格
- `1` 以上：减少 Selector 调用次数，提升速度

如果你的对话主题频繁变化，建议设置较小值。

### 5. 配置会话过滤

`session_filter_mode` 可选：

- `disabled`：全部会话生效
- `whitelist`：仅列表中的会话生效
- `blacklist`：列表中的会话不生效

`session_filter_list` 中可以填写群号、用户 ID 或会话 ID。

## 配置人格规则

`persona_rules` 是插件的核心配置。

每条规则需要填写：

- `rule_enabled`：是否启用该规则
- `persona_id`：从 AstrBot 已有人格中选择
- `provider_id`：从 AstrBot 已配置的对话模型条目中选择
- `persona_desc`：该人格是什么
- `scenario_desc`：什么情况下应切换到该人格

说明：

- `provider_id` 使用插件配置里的下拉框选择
- 该下拉框读取的是 AstrBot 当前已经配置好的对话模型条目
- 一个条目通常已经对应了某个具体模型配置，因此切换到该条目时，模型也会跟着切换
- 如果 `provider_id` 留空，则该人格只切 prompt，不强制切换对话模型

## 配置建议

### 人格描述

人格描述用于告诉 Selector “这个人格是谁”。

建议写清楚：

- 角色定位
- 说话风格
- 专长
- 禁止事项

### 场景描述

场景描述用于告诉 Selector “什么时候该用这个人格”。

建议写清楚：

- 触发条件
- 适用任务类型
- 不适用任务类型

场景描述越清晰，切换结果越稳定。

## 管理命令

插件提供以下命令：

- `/dp status`：查看当前插件状态和规则概要
- `/dp reload`：清空动态人格缓存
- `/dp enable`：启用插件
- `/dp disable`：禁用插件
- `/dp sessionid`：查看当前会话相关 ID

其中部分命令需要管理员权限。

## 重要限制

- 至少启用 2 条人格规则时，动态选择才有意义
- 如果当前会话被 AstrBot 的会话规则强制绑定人格，插件会默认跳过动态接管
- 如果 Selector 调用失败，插件会回退到默认规则或当前会话模型，不应阻塞消息主流程
- 本插件不会重构 AstrBot 的设置页面，它依赖 AstrBot 现有的人格与 Provider 配置能力

## 排错

### 没有发生人格切换

检查以下内容：

- 插件是否启用
- 是否至少启用了两条规则
- `persona_id` 是否在 AstrBot 中真实存在
- `scenario_desc` 是否足够明确
- 当前会话是否被过滤掉
- 当前会话是否被会话规则强制绑定了固定人格

### 没有发生模型切换

检查以下内容：

- 规则中的 `provider_id` 是否已选择
- 该 `provider_id` 是否对应 AstrBot 中已存在的对话模型条目
- 对应 Provider 是否可用
- AstrBot 当前 Provider 配置是否正常加载

### Selector 很慢

建议：

- 给 `selector_provider_id` 单独指定一个更快的模型
- 适当增大 `cache_ttl`
- 精简规则数量和描述长度

## 最小示例

两个常见规则示例：

- `default`：处理普通问题，绑定一个通用模型
- `雌小鬼`：处理明确的玩笑、挑逗、非严肃表达，绑定另一个模型

当用户输入普通问题时，插件会选择 `default`。

当用户输入带有明显调侃、暧昧或玩笑语气的内容时，插件会选择 `雌小鬼`。

## 版本说明

当前仓库中的实现重点是稳定的动态人格切换，以及与 AstrBot 已配置对话模型条目的联动。

如果后续 AstrBot 提供更标准的“插件配置直接选择模型列表”能力，可以在此基础上继续扩展，但当前版本以兼容 AstrBot 现有配置体系为主。
